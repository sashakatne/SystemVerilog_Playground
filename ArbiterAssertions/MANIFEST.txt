Purpose:
  Verify the 8-bit priority arbiter using SVA assertions injected
  non-intrusively via `bind`. The TB deliberately drives malformed stimulus
  (`r='z'`, `r='x'`, 257-cycle saturation, post-reset `r='x'`) so each
  assertion fires under the condition it is designed to catch, plus a
  256-vector exhaustive sweep that should produce no further fires.

  Assertions in assertions.sv (bound to Arbiter):
    1. a_valid_request           - r never has X/Z bits (except in reset)
    2. a_valid_grant             - g never has X/Z bits (except in reset)
    3. a_grant_in_one_cycle      - any request leads to a grant next cycle
    4. a_one_grant               - $onehot0(g) (never more than one grant)
    5. a_single_requestor_grant  - one-hot r leads to g == $past(r)
    6. a_grant_only_on_request   - g[i] is set only when $past(r[i]) was set
    7. a_grant_not_revoked       - if r is stable, g stays stable
    8. gen_request_limit[i]      - per-agent saturation counter (8 instances)

Source changes in this commit:
  assertions.sv:
    Added `int error_count = 0;` and wrapped every `$error(...)` else clause
    in begin/end with `error_count++;` so the top-level TB can read the
    bound assertion module's counter via a hierarchical reference.

  top.sv:
    After the test stimulus completes, the TB reads
    DUT.arbiter_assertions_inst.error_count and emits the repo-canonical
    verdict. Because the TB intentionally fires assertions, the verdict is
    framed as 'expected vs actual': pass when actual == 15 (the count this
    TB is designed to produce); fail otherwise. This converts an
    assertion-only testbench into one that emits the same final pass line
    every other module in the repo uses.

  arbiter.sv: NOT modified.

Caveats:
  - The verdict's expected count (15) is brittle: any change to the TB's
    stimulus or to the SVAs' disable/iff guards will shift it. If a future
    edit changes the stimulus, update the constant in top.sv.
  - The 'Errors: 15' in the simulator summary refers to assertion failure
    counts that Questa aggregates as errors, not compile/elab errors.
    The compile/elab Errors:0 line above proves the simulation itself was
    well-formed; the 15 'errors' are the captured SVA fires.
  - The `bind` directive lives in top.sv, but the bind target is the
    Arbiter module, so the assertions instance is reachable as
    DUT.arbiter_assertions_inst, not top.arbiter_assertions_inst.
