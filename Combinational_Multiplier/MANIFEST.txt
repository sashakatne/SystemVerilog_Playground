Purpose:
  Verify the structural NxN combinational multiplier built from a grid of
  multiplier_cell (full-adder + AND-gate) instances. At N=4 the exhaustive
  sweep is 16 x 16 = 256 multiplications, each compared against the
  reference `multiplicand * multiplier` predictor.

Source changes in this commit:
  combmultiplier.sv (NxN_multiplier generate block, lines ~55-90):
    The original generate used ternary selects with `m_out[i-1][j]`,
    `q_out[i][j-1]`, `pp[i-1][j+1]`, `carry[i-1][N-1]`, and `carry[i][j-1]`
    that referenced i-1 / j-1 even on row 0 / column 0 where those indices
    are out of bounds. Even though the ternary made those branches dead
    code, vopt still flagged 36 vopt-2576 (BSOB) and vopt-2696 (Index -1)
    warnings during optimization.
    The fix replaces the ternary cell-input expressions with a per-tile
    set of intermediate wires (`m_in`, `q_in`, `pp_in_w`, `carry_in_w`)
    driven by a generate-if chain so the i-1 / j-1 references never
    elaborate on the boundary tiles. The structural circuit is identical;
    only the elaboration shape changed. With the fix every vlog/vopt stage
    reports Errors: 0, Warnings: 0.

  combmultipliertb.sv lines 34, 36: '*** FAILED ***' / '*** PASSED ***' ->
    'Failed testbench' / 'No errors -- passed testbench'.
