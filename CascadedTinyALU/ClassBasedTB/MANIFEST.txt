Purpose:
  Verify the cascaded ALU via a class-based testbench (transaction, generator,
  driver, monitor, scoreboard, environment) communicating through mailboxes
  over a virtual interface. Exercises 33,600 randomized transactions
  (8 iterations x 4,200 ops) covering all 8 opcodes with two embedded
  covergroups (operation patterns + A/B corner crosses).

Source changes in this commit:
  run.do (rewritten):
    The original run.do invoked vsim three times in a row with no separate
    vopt step and never reached the coverage-save tail. It is now the
    canonical 6-step shape (catch{vdel-all} -> vlib work -> per-file
    vlog -source -lint -> vopt top -o top_optimized +acc +cover=sbfec+
    cascaded_alu(rtl). -> vsim -coverage -> run -all -> coverage save +
    vcover report). Compiled file order matches the original run.do.

  test.sv (lines 17-25):
    The test program no longer calls $stop. After env.run() returns it reads
    env.scb.no_trans and env.scb.errors, prints a one-line summary, then
    prints the canonical 'No errors -- passed testbench' or 'Failed
    testbench' verdict before calling $finish.

  environment.sv (line 62):
    The trailing $stop inside the run() task was removed so control returns
    to the caller (test.sv) for the verdict + $finish.

  scoreboard.sv:
    Added 'int errors;' counter alongside no_trans. On a result mismatch the
    scoreboard now increments errors. The per-transaction 'PASSED: ...'
    $display was removed so the transcript only emits FAILED prints if a
    mismatch occurs; the aggregate verdict in test.sv is the pass/fail
    signal. (This shrinks an otherwise multi-million-line transcript to a
    manageable size while preserving error visibility.)
